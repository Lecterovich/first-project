.gitconfig - файл в ~ директории.
git config --global user.name " "
git config --global user.email " "
git config --list - чтобы показать все настройки, которые Git найдёт
cat ~/.gitconfig -  показывает содержимое файла .gitconfig
Git - система контроля версий, которая помогает отслеживать изменения в проекте.

Команды:
cd - перемещение между директориями
rmdir - удаление директорий
rm - удаление файлов
mkdir - создание директорий
touch - создание файлов
git init - загитить папку (создать репозиторий)
rm -rf .git - разгитить папку
git add - сделать заготовку файла с изменениями
git add --all - подготовить к сохранию сразу все файлы
git commit - сохранить файл с изменениями
git push - отправить файл с изменениями на удалённый репозиторий
cp "файл" в "директория"- скопировать файл в другое место 
git status - проверить статус или состояние репозитория
pwd - путь к папке, где ты находишься в данный момент
ls - показать содержимое папки
ls -a - показать все скрытые папки и файлы, которые начинаются с символа .  
.. - перейти на уровень выше, в родительскую директорию
mv - перенести файл или папкув другое место
rm -r - удалить папку и всё, что в ней
Дважды tab - если лень набирать название папки или файла, можешь набрать только первые символы, и программа сама выведет тебе полное название этой папки или файла

*  * или _  _ - выделить текст курсивом
** ** или _ _  _ _ - выделить текст жирным шрифтом
# - самый крупный шрифт 
## - шрифт помельче
### - шрифт ещё мельче. Решётками создаются заголовки
--- - черта под заголовком или абзацем
Два пробела (или ..) или сочетание символов <br> - разрыв строки
Дважды enter - новый абзац
Один enter - текст сольётся в одну строку
~~ ~~ - зачеркнуть текст
Цифра с .  - создание нумерованных списков
* c пробелом или - с пробелом  - создание ненумерованных списков
[ ] ( ) - сделать ссылкой часть текста, заключив его в квадратные скобки и указав адрес в круглых
[ [ (   " ") - то же самое, только название в кавычках
